# 계산기 C 라이브러리 요구사항 명세서

본 문서는 정수 연산을 수행하는 C 라이브러리의 기능 요구사항 및 예외 처리 기준을 정의합니다.

## 1. 기본 원칙
- **반환값**: 모든 함수는 `int` 형의 에러 코드를 반환합니다.
    - `0`: 성공 (Success)
    - `음수`: 실패 (Error). Linux 커널 스타일의 에러 코드(예: `-EINVAL`, `-ERANGE`)를 준용합니다.
- **결과값**: 계산 결과는 함수의 마지막 인자인 포인터(`int *result`)를 통해 전달합니다.
- **오버플로우**: 연산 결과가 `int` 범위를 초과할 경우 에러로 처리합니다.

## 2. 에러 코드 정의
| 매크로명 | 값 | 설명 |
| :--- | :--- | :--- |
| `SUCCESS` | 0 | 연산 성공 |
| `E_INVALID_ARG` | -22 (`-EINVAL`) | 잘못된 인자 (예: 0으로 나누기, NULL 포인터) |
| `E_OVERFLOW` | -34 (`-ERANGE`) | 연산 결과가 범위를 벗어남 (Overflow/Underflow) |

## 3. 기능별 요구사항 세부 내역

| 기능 | 구분 | 입력 (a, b) | 기대 결과 (반환값, 결과값) | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **덧셈** | 정상 | 10, 20 | `0`, `30` | 일반적인 양수 합산 |
| | 정상 | -5, 3 | `0`, `-2` | 음수가 포함된 합산 |
| | 경계 | `INT_MAX`, 1 | `-ERANGE`, N/A | 양수 오버플로우 발생 시 에러 |
| | 경계 | `INT_MIN`, -1 | `-ERANGE`, N/A | 음수 언더플로우 발생 시 에러 |
| **뺄셈** | 정상 | 50, 20 | `0`, `30` | 일반적인 양수 뺄셈 |
| | 정상 | 10, 20 | `0`, `-10` | 결과가 음수인 뺄셈 |
| | 경계 | `INT_MIN`, 1 | `-ERANGE`, N/A | 언더플로우 발생 시 에러 |
| **곱셈** | 정상 | 10, 5 | `0`, `50` | 일반적인 곱셈 |
| | 정상 | -4, 3 | `0`, `-12` | 부호가 다른 두 수의 곱셈 |
| | 경계 | `INT_MAX`, 2 | `-ERANGE`, N/A | 오버플로우 발생 시 에러 |
| | 경계 | 0, 100 | `0`, `0` | 0을 곱하는 경우 |
| **나눗셈** | 정상 | 10, 2 | `0`, `5` | 일반적인 나눗셈 |
| | 정상 | 7, 3 | `0`, `2` | 소수점 이하는 버림 (정수 나눗셈) |
| | 에러 | 10, 0 | `-EINVAL`, N/A | **0으로 나누기 시도 시 에러** |
| | 경계 | `INT_MIN`, -1 | `-ERANGE`, N/A | `INT_MIN / -1`은 `INT_MAX`를 초과하므로 에러 |

## 4. API 설계 (예시)
```c
int add(int a, int b, int *result);
int sub(int a, int b, int *result);
int mul(int a, int b, int *result);
int div(int a, int b, int *result);
```
