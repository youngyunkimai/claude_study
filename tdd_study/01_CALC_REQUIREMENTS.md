# [실습 01] C 언어 계산기 TDD 요구사항 및 환경 설정

본 실습의 목표는 Gemini CLI와 함께 **Red-Green-Refactor** 사이클을 반복하며 사칙연산 계산기를 개발하는 것입니다.

## 1. 실습 환경
- **언어:** C (C11 표준 권장)
- **컴파일러:** `gcc`
- **빌드 도구:** `make` (선택 사항)
- **테스트 프레임워크:** `framework/test_framework.h` (매크로 기반 경량 프레임워크)

## 2. 프로젝트 구조 (예시)
실습을 위해 다음과 같은 구조로 시작합니다.
```text
tdd_practice/
└── 01_calc/
    ├── calculator.h      # 기능 선언
    ├── calculator.c      # 기능 구현
    └── test_calculator.c # 테스트 코드
```

## 3. 기능 요구사항
단계별로 다음 기능을 구현합니다.
1. **덧셈 (Add):** 두 정수를 입력받아 합을 반환한다.
2. **뺄셈 (Subtract):** 두 정수를 입력받아 차를 반환한다.
3. **곱셈 (Multiply):** 두 정수를 입력받아 곱을 반환한다.
4. **나눗셈 (Divide):**
   - 두 정수를 입력받아 몫을 반환한다.
   - **예외 처리:** 0으로 나눌 경우 특정 에러 값(예: 0 또는 에러 코드)을 반환하거나 처리한다.

## 4. 실습 규칙
- Gemini에게 한 번에 모든 코드를 짜달라고 하지 마세요.
- **테스트 코드 1개 작성 -> 컴파일 실패(Red) -> 최소 구현(Green) -> 정리(Refactor)** 순서를 엄격히 지킵니다.
